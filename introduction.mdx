MLX AudioGen is an on-device AI audio generation toolkit built for Apple Silicon. It brings together three state-of-the-art models — MusicGen, Stable Audio Open, and Demucs v4 — and runs all inference locally through Apple's Metal GPU using the MLX framework. Whether you're a musician sketching ideas, a developer building audio tools, or a producer working inside a DAW, you get fast, private, unlimited generation without touching a cloud API.

## What makes it different

Every other AI audio tool sends your prompts and audio to a remote server. MLX AudioGen runs entirely on your Mac. Inference happens on the Metal GPU via the [MLX framework](https://github.com/ml-explore/mlx), which means:

- **No cloud dependency** — works offline, no API keys or accounts required
- **No usage limits** — generate as much as you want
- **Private by default** — your prompts and audio never leave your machine
- **Full-stack access** — CLI, HTTP server, React web UI, VST3/AU plugin, and Max for Live all included

## Supported models

| Model family | Variants | Output | Sample rate |
|---|---|---|---|
| MusicGen | small, medium, large (mono + stereo) | WAV | 32 kHz |
| MusicGen Melody | base, large | Mono WAV | 32 kHz |
| MusicGen Style | base | Mono WAV | 32 kHz |
| Stable Audio Open | small, 1.0 | Stereo WAV | 44.1 kHz |
| Demucs v4 | htdemucs, htdemucs_6s | 4–6 stems | 44.1 kHz |

Pre-converted MLX weights for all variants are published on [HuggingFace](https://huggingface.co/jasonvassallo) and auto-download on first use.

## System requirements

<Note>
MLX AudioGen requires an Apple Silicon Mac. The MLX framework uses Metal GPU acceleration, which is only available on M1 or later chips. Intel Macs and non-Apple hardware are not supported.
</Note>

| Requirement | Minimum |
|---|---|
| Hardware | Apple Silicon (M1 or later) |
| OS | macOS |
| Python | 3.11 or later |
| Disk | Varies by model — small models ~500 MB, large models ~4 GB |
