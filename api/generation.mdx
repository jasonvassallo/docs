---
title: "Submit audio generation jobs with POST /api/generate"
description: "Submit text-to-audio generation jobs, poll status with real-time progress, and download WAV, AIFF, FLAC, or MIDI output from the MLX AudioGen API."
---

Audio generation in MLX AudioGen is asynchronous. You submit a request to `POST /api/generate`, receive a `job_id` immediately, then poll `GET /api/status/{id}` until the job is complete. Once done, you download the result from `GET /api/audio/{id}`. If you requested MIDI output, it is available at `GET /api/midi/{id}`.

---

## POST /api/generate

Submit a generation request. The server enqueues the job and returns immediately — it does not wait for generation to finish.

### Request body

<ParamField body="model" type="string" required>
  Model to use for generation. Must match `^(musicgen|stable_audio|moss_sfx|lyria-002|lyria-3-pro-preview|lyria-3-clip-preview)$`.

  - `"musicgen"` — autoregressive music, 32 kHz, mono or stereo (local)
  - `"stable_audio"` — diffusion-based music, 44.1 kHz stereo (local)
  - `"moss_sfx"` — MOSS-SoundEffect text-to-sound-effect, 24 kHz mono (local)
  - `"lyria-002"` — **cloud** (Google Vertex AI), Lyria 2, ~30 s 48 kHz stereo
  - `"lyria-3-pro-preview"` / `"lyria-3-clip-preview"` — **cloud** (Google Vertex AI), Lyria 3 **preview**

  Default: `"musicgen"`

  <Warning>
    The `lyria-*` values run on **Google Vertex AI, not on your machine** — selecting one sends your prompt to Google Cloud (egress) and returns SynthID-watermarked audio. They are only accepted when the server has `GOOGLE_CLOUD_PROJECT` set and the `[cloud]` extra installed (authentication is via Application Default Credentials — no API key). If Lyria is not configured, the job fails with an error explaining how to enable it. The `lyria-3-*` ids are preview/experimental. See [Cloud generation with Lyria](#cloud-generation-with-lyria).
  </Warning>
</ParamField>

<ParamField body="prompt" type="string" required>
  Text description of the audio you want to generate. Minimum 1 character, maximum 5000 characters. The T5 tokenizer truncates inputs beyond 512 tokens.
</ParamField>

<ParamField body="negative_prompt" type="string">
  Describes characteristics to avoid in the output. Maximum 5000 characters. Used by `stable_audio` and by Lyria 2 (`lyria-002`); ignored by `musicgen` and by the Lyria 3 preview models (`lyria-3-*`).

  Default: `""`
</ParamField>

<ParamField body="seconds" type="float">
  Duration of the generated audio in seconds. Must be between `0.1` and `300`.

  Default: `5.0`

  <Note>
    MusicGen produces reliable results up to ~30 seconds; Stable Audio up to ~47 seconds. Very long durations may produce lower-quality output.
  </Note>
</ParamField>

<ParamField body="seed" type="integer | null">
  Random seed for reproducibility. Pass an integer to get deterministic output. `null` uses a random seed each time. For Lyria, `seed` applies to Lyria 2 (`lyria-002`) only and is ignored by the Lyria 3 preview models.

  Default: `null`
</ParamField>

<ParamField body="output_mode" type="string">
  Controls what output the job produces. Must match `^(audio|midi|both)$`.

  - `"audio"` — WAV/AIFF/FLAC file only
  - `"midi"` — transcribes the generated audio to MIDI (audio is still generated internally)
  - `"both"` — audio file and MIDI file

  Default: `"audio"`
</ParamField>

#### MusicGen parameters

<ParamField body="temperature" type="float">
  Sampling temperature. Higher values produce more creative, varied output; lower values are more deterministic. Must be greater than `0`.

  Default: `1.0`
</ParamField>

<ParamField body="top_k" type="integer">
  Number of top-k token candidates considered at each sampling step. Must be `≥ 1`.

  Default: `250`
</ParamField>

<ParamField body="guidance_coef" type="float">
  Classifier-free guidance scale. Higher values make output follow the prompt more closely. Must be `≥ 0`.

  Default: `3.0`
</ParamField>

<ParamField body="melody_path" type="string | null">
  Absolute path to a WAV file used for melody conditioning (melody model variants only). The pipeline extracts a 12-bin chromagram and uses it as additional conditioning. Maximum 1024 characters. Must not contain `..`.

  Default: `null`
</ParamField>

<ParamField body="style_audio_path" type="string | null">
  Absolute path to a WAV file used for style conditioning (style model variant only). MERT extracts timbre features from the reference to guide generation. Maximum 1024 characters. Must not contain `..`.

  Default: `null`
</ParamField>

<ParamField body="style_coef" type="float">
  Dual-CFG text influence coefficient for style conditioning. Controls how strongly the text prompt influences output relative to the style reference. Must be `≥ 0`.

  Default: `5.0`
</ParamField>

<ParamField body="stream" type="boolean">
  Opt in to a progressive PCM preview of the audio while it generates. Only valid for `model: "musicgen"` with `mode: "generate"` (the default mode) — combining `stream: true` with `mode: "continue"`, `mode: "inpaint"`, or any other model fails validation with `HTTP 422`.

  When `true`, connect to [`GET /api/stream/{id}`](#get-apistreamid) after submitting the job to hear audio as it's produced, well before the final WAV is ready. Streaming has no effect on the final result: the WAV, MIDI, detected key, and quality score are identical whether or not you ever open a stream connection, and whether or not the stream reader disconnects partway through.

  Default: `false`
</ParamField>

#### Stable Audio parameters

<ParamField body="steps" type="integer">
  Number of diffusion steps. More steps produce higher quality at the cost of speed. Must be between `1` and `1000`.

  Default: `8`
</ParamField>

<ParamField body="cfg_scale" type="float">
  CFG guidance scale for Stable Audio. Must be `≥ 0`.

  Default: `6.0`
</ParamField>

<ParamField body="sampler" type="string">
  ODE sampler to use. Must match `^(euler|rk4)$`.

  Default: `"euler"`
</ParamField>

<ParamField body="reference_audio_path" type="string | null">
  Absolute path to a reference audio file for audio-to-audio generation (Stable Audio only). The model conditions on the reference to produce variations. Maximum 1024 characters. Must not contain `..`.

  Default: `null`
</ParamField>

<ParamField body="reference_strength" type="float">
  How strongly the reference audio influences output. `0.0` ignores the reference; `1.0` stays close to it. Must be between `0.0` and `1.0`.

  Default: `0.7`
</ParamField>

#### MOSS-SoundEffect parameters

These parameters apply when `model` is `"moss_sfx"`. They are ignored by other models.

<ParamField body="top_p" type="float">
  Nucleus-sampling probability mass for MOSS-SoundEffect. Must be greater than `0.0` and at most `1.0`.

  Default: `0.6`
</ParamField>

<ParamField body="repetition_penalty" type="float">
  Penalizes repeated tokens during MOSS-SoundEffect sampling to avoid looping artifacts. Must be between `1.0` and `2.0`.

  Default: `1.2`
</ParamField>

<Note>
  MOSS-SoundEffect also reuses the shared `temperature` and `top_k` fields, but with model-specific tuning. Their schema defaults (`temperature: 1.0`, `top_k: 250`) are tuned for MusicGen and degrade MOSS output. If you **omit** `temperature` and/or `top_k` from a `moss_sfx` request, the server substitutes MOSS's recommended defaults — `temperature: 1.5` and `top_k: 50`. Any value you set explicitly is used verbatim, even if it equals a MusicGen default, so to get MOSS-tuned sampling either send `1.5`/`50` or leave those fields out entirely.
</Note>

<ParamField body="lora" type="string | null">
  Name or path of a LoRA adapter to apply to the MusicGen pipeline. Maximum 200 characters.

  Default: `null`
</ParamField>

#### Continuation & inpaint parameters (Phase 10i)

<Info>
  These fields extend an **existing** generation instead of starting a fresh one: `mode: "continue"` appends new audio after a source clip; `mode: "inpaint"` regenerates a region inside it, conditioned on the audio before that region and crossfaded back in at both edges. See the [Continue & inpaint guide](/guides/continue-and-inpaint) for the full walkthrough, the two web UI entry points, and its v1 scope.
</Info>

<ParamField body="mode" type="string">
  Generation mode. Must match `^(generate|continue|inpaint)$`.

  - `"generate"` — a fresh, independent generation
  - `"continue"` — append `seconds` of new audio that follows the source
  - `"inpaint"` — regenerate `region_start`..`region_end` inside the source, conditioned on the LEFT context only (MusicGen is autoregressive — there is no "future" audio to condition the right edge on)

  Default: `"generate"`

  <Warning>
    `"continue"`/`"inpaint"` are only supported for `model: "musicgen"`, and only for **plain mono, 4-codebook variants** — melody, style, and stereo variants are rejected (the shared 4-quantizer EnCodec cannot decode stereo's 8 codebooks). They also cannot be combined with `melody_path`, `style_audio_path`, or `reference_audio_path`.
  </Warning>
</ParamField>

<ParamField body="source_job_id" type="string | null">
  The 8-character job ID of the source generation to continue/inpaint, while that job is still in server memory (jobs are cleaned up ~5 minutes after completion). Must match `^[a-f0-9]{8}$`. Provide exactly one of `source_job_id` or `source_path` when `mode` is `"continue"` or `"inpaint"`.

  Default: `null`
</ParamField>

<ParamField body="source_path" type="string | null">
  A server-allocated path returned by [`POST /api/source-upload`](#post-apisource-upload), used once the source job has already been cleaned up. Maximum 1024 characters.

  Default: `null`
</ParamField>

<ParamField body="context_seconds" type="float">
  Seconds of the source's tail used as conditioning context. Must be between `1.0` and `20.0`.

  Default: `10.0`

  <Note>
    MusicGen's autoregressive decoder caps `context_seconds + seconds` (continue) or `context_seconds + (region_end - region_start)` (inpaint) at **39.94 seconds** — its 2000-position limit minus 3 for the 4-codebook delay pattern. A request over this limit fails validation with `HTTP 422`.
  </Note>
</ParamField>

<ParamField body="region_start" type="float | null">
  Start of the region to regenerate, in seconds from the start of the source (`mode: "inpaint"` only). Must be between `0.0` and `300.0`, and less than `region_end`.

  Default: `null`
</ParamField>

<ParamField body="region_end" type="float | null">
  End of the region to regenerate, in seconds (`mode: "inpaint"` only). Must be between `0.0` and `300.0`. The region (`region_end - region_start`) cannot exceed `30` seconds.

  Default: `null`
</ParamField>

<ParamField body="crossfade_ms" type="float">
  Length of the equal-power crossfade used to splice new audio back into the source, in milliseconds. Must be between `10.0` and `500.0`.

  Default: `60.0`
</ParamField>

<ParamField body="extension_only" type="boolean">
  When `mode` is `"continue"`, return only the newly generated audio instead of the full source-plus-continuation result. Requires `mode: "continue"` — invalid combined with `"generate"` or `"inpaint"`.

  Default: `false`
</ParamField>

### Response

<ResponseField name="id" type="string">
  Job ID — an 8-character hex string (e.g., `"a3f1c2d4"`). Use this to poll status and download output.
</ResponseField>

<ResponseField name="status" type="string">
  Always `"queued"` on a successful submission.
</ResponseField>

### Example

<CodeGroup>

```bash curl
curl -X POST http://localhost:8420/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "musicgen",
    "prompt": "upbeat jazz with piano and brushed drums",
    "seconds": 15,
    "temperature": 1.0,
    "top_k": 250,
    "guidance_coef": 3.0,
    "output_mode": "both",
    "seed": 42
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8420/api/generate",
    json={
        "model": "musicgen",
        "prompt": "upbeat jazz with piano and brushed drums",
        "seconds": 15,
        "temperature": 1.0,
        "top_k": 250,
        "guidance_coef": 3.0,
        "output_mode": "both",
        "seed": 42,
    },

data = response.json()

print(f"Job submitted: {job_id}")
```

</CodeGroup>

## POST /api/source-upload

Upload an audio file to use as a continuation/inpaint source once the original generation job has already been cleaned up (completed jobs are kept in server memory for only ~5 minutes). Returns a server-allocated path — pass it back as `source_path` on `POST /api/generate`.

<Info>
  You usually don't need this: while the source job is still in memory, pass `source_job_id` instead and skip the upload entirely. The web UI's **Extend…** and **↻ Regenerate section** actions try `source_job_id` first and only upload automatically as a fallback.
</Info>

### Request

Multipart form upload with a single `file` field. Allowed extensions: `.wav`, `.mp3`, `.flac`, `.ogg`, `.aac`, `.m4a`, `.aiff`, `.aif`. Maximum size 128 MB.

### Response

<ResponseField name="path" type="string">
  Server-allocated temporary file path. Pass this back as `source_path` on `POST /api/generate`. The upload is deleted automatically once the generation job that consumes it finishes; orphaned uploads (rejected request, crash) are swept on server startup.
</ResponseField>

Returns `HTTP 400` for an unsupported file extension or an empty upload. Returns `HTTP 413` if the file exceeds 128 MB.

### Example

```bash curl
curl -X POST http://localhost:8420/api/source-upload \
  -F "file=@source.wav"
```

```json Response
{
  "path": "/Users/you/.mlx-audiogen/source_tmp/tmpabc123.wav"
}
```

## Cloud generation with Lyria

Lyria is the only **cloud** model you can use with MLX AudioGen. It runs on **Google Vertex AI**, not on your Mac. It is also **opt-in** — your server only accepts the `lyria-*` model values once you configure it. You drive Lyria exactly like a local model. Submit your request to `POST /api/generate`, poll `GET /api/status/{id}`, then download your audio from `GET /api/audio/{id}`. Each Lyria generation is a single blocking cloud round-trip (batch). It runs in the same job queue as your local generations, with no streaming.

<Warning>
**Cloud egress + watermarking.** Selecting a `lyria-*` model sends your prompt to Google Cloud and returns SynthID-watermarked audio. This is unlike every other MLX AudioGen model, which runs entirely on-device. Enable Lyria only if you are comfortable with prompt egress to Google.
</Warning>

### Enabling Lyria on the server

The server only offers Lyria when **both** of these are true:

- The optional `[cloud]` extra is installed: `pip install "mlx-audiogen[cloud]"` (adds `google-genai` + `google-auth`).
- The `GOOGLE_CLOUD_PROJECT` environment variable is set to your Google Cloud project id. Optionally set `GOOGLE_CLOUD_LOCATION` (defaults to `us-central1`; only affects `lyria-002`).

You authenticate with **Application Default Credentials (ADC)** — there is **no API key**. The server uses whatever ADC / Workload Identity Federation login you have configured on the host. For local use, run `gcloud auth application-default login`. When you leave `GOOGLE_CLOUD_PROJECT` unset, the Lyria models do not appear in [`GET /api/models`](/api/models). A `lyria-*` generation request then fails with an error explaining how to enable it.

### Model ids and supported fields

| Model id | Lyria version | Status | `negative_prompt` | `seed` | Output |
|---|---|---|---|---|---|
| `lyria-002` | Lyria 2 | Stable (verified) | Yes | Yes | ~30 s, 48 kHz stereo |
| `lyria-3-pro-preview` | Lyria 3 Pro | **Preview** | No | No | text input only |
| `lyria-3-clip-preview` | Lyria 3 Clip | **Preview** | No | No | text input only |

Lyria returns fixed-length clips (~30 s for Lyria 2). So `seconds` does not control the cloud generation length. Instead, the server uses `seconds` only to trim the returned audio. Lyria also ignores the MusicGen/Stable Audio parameters you may be used to (`temperature`, `steps`, `cfg_scale`, and so on).

### Example

```bash curl
curl -X POST http://localhost:8420/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "lyria-002",
    "prompt": "warm analog synthwave with a driving bassline",
    "negative_prompt": "vocals, distortion",
    "seed": 42
  }'
```

## GET /api/status/{id}

Poll the status of a generation job. Call this repeatedly until `status` is `"done"` or `"error"`.

### Path parameters

<ParamField path="id" type="string" required>
  Job ID returned by `POST /api/generate`.
</ParamField>

### Response

<ResponseField name="id" type="string">
  Job ID.
</ResponseField>

<ResponseField name="status" type="string">
  Current job state: `"queued"`, `"running"`, `"done"`, or `"error"`.
</ResponseField>

<ResponseField name="progress" type="float">
  Generation progress from `0.0` to `1.0`. Updated in real time while the job is running.
</ResponseField>

<ResponseField name="model" type="string">
  Model that was requested.
</ResponseField>

<ResponseField name="prompt" type="string">
  Prompt from the original request.
</ResponseField>

<ResponseField name="seconds" type="float">
  Requested duration.
</ResponseField>

<ResponseField name="created_at" type="float">
  Unix timestamp when the job was created.
</ResponseField>

<ResponseField name="completed_at" type="float | null">
  Unix timestamp when the job finished. `null` while still running.
</ResponseField>

<ResponseField name="sample_rate" type="integer | null">
  Sample rate of the generated audio. `null` until the job completes. MusicGen: 32000. Stable Audio: 44100. MOSS-SoundEffect: 24000. Lyria: 48000 (read from the returned WAV).
</ResponseField>

<ResponseField name="has_midi" type="boolean">
  Whether a MIDI file is available for download on this job.
</ResponseField>

<ResponseField name="starred" type="boolean">
  Whether the job has been starred as a favorite.
</ResponseField>

<ResponseField name="mode" type="string">
  Generation mode used for this job: `"generate"`, `"continue"`, or `"inpaint"` (Phase 10i). See [Continuation & inpaint parameters](#continuation-inpaint-parameters-phase-10i).

  Default: `"generate"`
</ResponseField>

<ResponseField name="detected_key" type="string | null">
  Detected musical key of the generated audio, e.g. `"A minor"` or `"F# major"`. Detection runs for every generation. `"unknown"` means detection ran but found no clear key (silence or atonal material); `null` means the key was not recorded (a job from before key detection existed, or a detector failure — the generation is unaffected either way).
</ResponseField>

<ResponseField name="key_confidence" type="float | null">
  Confidence of the detected key from `0.0` to `1.0`. `null` whenever `detected_key` is `null`.
</ResponseField>

<ResponseField name="quality_overall" type="float | null">
  Automated quality score from `0.0` (poor) to `1.0` (clean), computed after the job completes. Only present for generations that used a LoRA adapter; `null` otherwise.
</ResponseField>

<ResponseField name="quality_flags" type="string[] | null">
  Human-readable quality issues detected in the output (for example `"noisy"`, `"clipping"`, `"no-dynamics"`, `"spectral-imbalance"`). Empty array when no issues are found. Only present for LoRA-active generations; `null` otherwise.
</ResponseField>

<ResponseField name="error" type="string | null">
  Error message if `status` is `"error"`. `null` otherwise.
</ResponseField>

<Note>
  **Automated quality metrics.** When a generation uses a LoRA adapter, MLX AudioGen runs a lightweight, pure-numpy health analysis on the output and reports it as `quality_overall` (0–1) plus `quality_flags`. The same metrics underpin the flywheel's self-correction capability: when a retrained adapter version scores worse than the version before it, the flywheel keeps the older, better version active instead of promoting a regression. See [Flywheel auto-retrain](#flywheel-auto-retrain).
</Note>

<Note>
  **Key detection.** Every generation reports a detected musical key in `detected_key` (with `key_confidence`), estimated from the audio's chroma profile via the Krumhansl–Kessler key profiles. It is best-effort telemetry — a detection failure leaves the key unset and never affects the generated audio. In the web UI, the detected key appears as a badge on each history entry.
</Note>

### Example

<CodeGroup>

```bash curl
curl http://localhost:8420/api/status/a3f1c2d4
```

```python Python (polling loop)
import time
import requests

def wait_for_job(job_id: str, poll_interval: float = 1.0) -> dict:
    url = f"http://localhost:8420/api/status/{job_id}"
    while True:
        resp = requests.get(url)
        resp.raise_for_status()
        data = resp.json()
        status = data["status"]
        progress = data.get("progress", 0.0)
        print(f"Status: {status} ({progress:.0%})")
        if status == "done":
            return data
        if status == "error":
            raise RuntimeError(f"Generation failed: {data.get('error')}")
        time.sleep(poll_interval)

result = wait_for_job("a3f1c2d4")
```

</CodeGroup>

```json Response
{
  "id": "a3f1c2d4",
  "status": "done",
  "progress": 1.0,
  "model": "musicgen",
  "prompt": "upbeat jazz with piano and brushed drums",
  "seconds": 15.0,
  "created_at": 1715000000.0,
  "completed_at": 1715000023.4,
  "sample_rate": 32000,
  "has_midi": true,
  "starred": false,
  "detected_key": "A minor",
  "key_confidence": 0.78,
  "quality_overall": 0.86,
  "quality_flags": [],
  "error": null
}
```

## GET /api/audio/{id}

Download the generated audio file. The job must have `status: "done"` before calling this endpoint.

### Path parameters

<ParamField path="id" type="string" required>
  Job ID from `POST /api/generate`.
</ParamField>

### Query parameters

<ParamField query="fmt" type="string">
  Output format. One of `wav`, `aiff`, or `flac`.

  Default: `"aiff"`
</ParamField>

### Response

Returns the audio file as a binary response with the appropriate `Content-Type` header:

| Format | Content-Type |
|---|---|
| `wav` | `audio/wav` |
| `aiff` | `audio/aiff` |
| `flac` | `audio/flac` |

The `Content-Disposition` header is set to `attachment; filename="<job_id>.<ext>"`.

Returns `HTTP 202` (with an error message) if the job is still in progress. Returns `HTTP 404` if the job ID does not exist.

### Example

<CodeGroup>

```bash curl
curl -o output.wav "http://localhost:8420/api/audio/a3f1c2d4?fmt=wav"
```

```python Python
import requests

resp = requests.get(
    "http://localhost:8420/api/audio/a3f1c2d4",
    params={"fmt": "wav"},
)
resp.raise_for_status()
with open("output.wav", "wb") as f:
    f.write(resp.content)
print("Audio saved to output.wav")
```

</CodeGroup>

Download the generated MIDI file. Only available when `output_mode` was `"midi"` or `"both"` in the original generation request.

### Path parameters

<ParamField path="id" type="string" required>
  Job ID from `POST /api/generate`.
</ParamField>

Returns the MIDI file as `audio/midi` with `Content-Disposition: attachment; filename="<job_id>.mid"`.


## GET /api/stream/\{id\}

Stream a progressive PCM preview of an in-flight MusicGen generation. Only available when the original `POST /api/generate` request had `stream: true` and `mode: "generate"` — otherwise the job never has a stream to serve.

<Info>
  This endpoint exists purely for **preview**. The final result is completely unaffected by streaming: whether you connect, when you connect, or whether your connection drops midway, the job still produces the same WAV, MIDI, detected key, and quality score. A slow or disconnected reader never blocks or fails generation.
</Info>

### Path parameters

<ParamField path="id" type="string" required>
  Job ID from `POST /api/generate`. The originating request must have had `stream: true`.
</ParamField>

### Response

A `200` response streams binary frames as chunked HTTP, `Content-Type: application/octet-stream`. Each frame has the same three-part layout:

| Field | Type | Description |
|---|---|---|
| `type` | `u8` | `1` = header, `2` = PCM chunk, `3` = end |
| `len` | `u32` (little-endian) | Byte length of `payload` |
| `payload` | bytes | Frame body — see below |

| Type | Payload |
|---|---|
| `1` (header) | UTF-8 JSON: `{"format": "s16le", "sample_rate": <int>, "channels": <int>}` — sent once, first |
| `2` (PCM chunk) | Raw signed 16-bit little-endian PCM samples, interleaved by channel |
| `3` (end) | UTF-8 JSON: `{"done": <bool>, "error": <string \| null>, "truncated": <bool>}` — sent once, last |

The stream always starts with exactly one header frame, is followed by zero or more PCM chunk frames as audio becomes available, and ends with exactly one end frame. `truncated: true` in the end frame means the preview stopped early (for example, the replay buffer's 64 MB cap was reached) — it does **not** mean the final generation was cut short.

Only one client may read a job's stream at a time — a second concurrent connection gets `HTTP 409`. Reconnecting (after a disconnect, or as a fresh client) always replays from frame 0, so a late-joining client sees the full preview from the start, not just what's left. If you request a stream that doesn't exist (unknown job, or `stream` wasn't set on the original request), you get `HTTP 404`.

This endpoint is exempt from the generation rate limit and uses the general rate-limit bucket instead.

### Example

```bash curl
curl -N http://localhost:8420/api/stream/a3f1c2d4 -o preview.raw
```

```python Python
import json
import struct

import requests

def read_stream(job_id: str):
    url = f"http://localhost:8420/api/stream/{job_id}"
    with requests.get(url, stream=True) as resp:
        resp.raise_for_status()
        buf = resp.raw
        while True:
            header = buf.read(5)
            if not header:
                break
            frame_type, length = struct.unpack("<BI", header)
            payload = buf.read(length)
            if frame_type == 1:
                print("header:", json.loads(payload))
            elif frame_type == 2:
                pass  # payload is raw s16le PCM — write/play it
            elif frame_type == 3:
                print("end:", json.loads(payload))
                break

read_stream("a3f1c2d4")
```

## Flywheel auto-retrain

The flywheel turns the generations you star into improved LoRA adapters. Each time you star a generation made with an adapter, the flywheel counts it; once the count crosses the adapter's retrain threshold, a retrain is triggered automatically. You can also trigger one manually.

A retrain runs end to end on a background thread: build a fresh training set from your library and starred generations → train a new adapter version → evaluate it with the [automated quality metrics](#get-apistatusid) → and gate the result. If the new version scores worse than the version before it, the flywheel keeps the previous version active instead of promoting the regression. This is what makes the quality gate self-correcting — only versions that hold up are kept.

### POST /api/flywheel/retrain/\{name\}

Manually trigger a retrain for an adapter. Returns immediately with a retrain job ID; poll `GET /api/flywheel/retrain/status/{name}` for live progress. Only one retrain runs at a time across the server.

<ParamField path="name" type="string" required>
  Adapter name. Must not contain `..`, `/`, or `\`.
</ParamField>

<ResponseField name="id" type="string">
  Retrain job ID. Present when `status` is `"started"`.
</ResponseField>

<ResponseField name="status" type="string">
  `"started"` when the retrain began, or `"skipped"` if a retrain (or a user training run) is already in progress.
</ResponseField>

<ResponseField name="adapter" type="string">
  The adapter name.
</ResponseField>

<ResponseField name="reason" type="string">
  Why the retrain was skipped (e.g. `"already running"` or `"training in progress"`). Present only when `status` is `"skipped"`.
</ResponseField>

```bash
curl -X POST http://localhost:8420/api/flywheel/retrain/my-lofi-lora
```

```json Response
{
  "id": "b7e2",
  "status": "started",
  "adapter": "my-lofi-lora"
}
```

### GET /api/flywheel/retrain/status/\{name\}

Poll the live status of an adapter's retrain.

<ParamField path="name" type="string" required>
  Adapter name. Must not contain `..`, `/`, or `\`.
</ParamField>

<ResponseField name="adapter" type="string">
  The adapter name.
</ResponseField>

<ResponseField name="phase" type="string">
  Current phase: `"idle"` (never retrained), `"building"`, `"training"`, `"evaluating"`, then a terminal phase — `"done"` (new version accepted), `"regressed"` (new version scored worse, previous version kept active), `"error"`, or `"skipped"`.
</ResponseField>

<ResponseField name="version" type="string | null">
  The new adapter version created during this retrain (e.g. `"v3"`). Present once a version exists.
</ResponseField>

<ResponseField name="accepted" type="boolean | null">
  Whether the new version was promoted to active. `false` when it regressed and the previous version was kept.
</ResponseField>

<ResponseField name="mean_score" type="float | null">
  Mean quality score of the new version's evaluation set (`0.0`–`1.0`). Present in terminal phases.
</ResponseField>

<ResponseField name="error" type="string | null">
  Error message if `phase` is `"error"`. `null` otherwise.
</ResponseField>

```bash
curl http://localhost:8420/api/flywheel/retrain/status/my-lofi-lora
```

```json Response
{
  "adapter": "my-lofi-lora",
  "phase": "done",
  "version": "v3",
  "accepted": true,
  "mean_score": 0.84,
  "error": null
}
```
